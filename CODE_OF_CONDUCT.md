# Code of Conduct

## Our Pledge

We want AgentForge to be a welcoming, respectful, and constructive project for everyone.

Contributors, maintainers, and community members are expected to:

- be respectful in language and tone
- assume good intent while still giving clear technical feedback
- focus criticism on ideas, code, and behavior, not people
- help create an environment where people can ask questions safely

## Expected Behavior

- Use welcoming and inclusive language.
- Be respectful of different viewpoints and experience levels.
- Accept constructive feedback gracefully.
- Prioritize collaboration, clarity, and kindness.
- Show empathy when reviewing, discussing, or debugging.

## Unacceptable Behavior

- Harassment, intimidation, or discrimination
- Personal attacks, insults, or derogatory language
- Publishing private information without permission
- Sustained disruptive behavior in issues, pull requests, or discussions
- Deliberately hostile or bad-faith participation

## Scope

This Code of Conduct applies in:

- GitHub issues
- pull requests
- code review discussions
- project documentation and community spaces related to AgentForge

## Enforcement

Project maintainers may remove, edit, or reject contributions, comments, commits, issues, or other interactions that violate this Code of Conduct.

## Reporting

If you experience or witness unacceptable behavior, please contact the maintainers privately through GitHub profile contact options when possible.

If a private contact path is not available, open a public issue only for non-sensitive moderation requests.

## Attribution

This document is informed by common open source community standards, including the Contributor Covenant.
